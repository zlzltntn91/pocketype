# /mistake — 실수 기록

힌트: $ARGUMENTS

너는 오케스트레이터다. 아래 절차를 따라 실수를 파악하고 CLAUDE.md에 기록한다.

## Step 1 — 컨텍스트 수집

다음을 병렬로 읽는다:
- `/Users/zlamstn/Documents/tutorial-hannes/CLAUDE.md` — 기존 실수 목록 확인 (중복 방지)
- `/Users/zlamstn/Documents/tutorial-hannes/.claude/learnings.md` — 최근 학습 내역
- `/Users/zlamstn/Documents/tutorial-hannes/.claude/logs/` 디렉터리가 있으면 최근 로그 파일 1~2개

힌트(`$ARGUMENTS`)가 있으면 해당 키워드로 Grep해 관련 소스/테스트 파일도 확인한다.

## Step 2 — 실수 특정

수집한 컨텍스트와 현재 대화 이력을 바탕으로 다음을 판단한다:
- **무엇을** 잘못했는가 (잘못된 판단, 누락, 회귀 등)
- **왜** 발생했는가 (프로세스 구멍, 판단 기준 부재 등)
- **교훈** — 다음에 같은 실수를 막으려면 어떤 기준을 추가해야 하는가

판단이 모호하거나 힌트가 없어 실수를 특정하기 어려우면 `AskUserQuestion`으로 사용자에게 확인한다.
추측으로 기록하지 않는다.

## Step 3 — 중복 확인

CLAUDE.md의 기존 실수 목록과 비교해 실질적으로 동일한 내용이 이미 있으면 기록하지 않고 사용자에게 알린다.

## Step 4 — CLAUDE.md 업데이트 (두 섹션 모두)

날짜는 시스템 컨텍스트의 `오늘 날짜`에서 가져온다 (없으면 Bash로 `date +%Y-%m-%d` 실행).

### 4-A. `## 과거 실수 (참조용)` 섹션 마지막에 추가

형식:
```
- **{날짜} {작업 이름}**: {어떤 판단을 했고 어떤 결과가 나왔는지 구체적으로}. **교훈**: {다음에 같은 실수를 막기 위한 구체적 기준}.
```

### 4-B. `## 중복 실수 방지` 섹션 업데이트

**테이블 행 추가** — `| 수정일자 | 수정사유 | 수정항목 명세 |` 테이블 마지막에 한 행 추가:
```
| {날짜} | {어떤 실수 때문에} | {CLAUDE.md 또는 에이전트 프롬프트에서 무엇을 어떻게 바꿨는지} |
```

**`### Reviewer 추가 체크리스트` 또는 관련 섹션 갱신** — 실수가 특정 역할(Reviewer, Tester, Implementer 등)의 판단 기준 부재에서 비롯됐다면, 해당 체크리스트에 구체적 검증 항목을 추가한다. 오케스트레이터 판단 기준 문제라면 관련 섹션(스킵 조건, 직접 수행 허용 등)을 보강한다.

## Step 5 — 완료 보고

추가/수정한 내용 전문을 사용자에게 보여준다.
